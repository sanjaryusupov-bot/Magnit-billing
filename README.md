# MCOS — дашборд «Статус по отгрузкам с РЦ» (Streamlit)

## Что внутри

3 вкладки:

1. **📅 Отгрузки по дням** — День / Кол-во шт / Итого сумма (график + таблица) с переходом
   в детали по конкретному дню: Дата, Магазин, Кол-во шт, Сумма.
2. **🗺️ Сводная (Город / Транспорт / Рейс)** — Город, Транспорт, Рейс, Итого сумма, Кол-во шт.
   Город берётся из листа «Статус по заказам» (по номеру заказа), с фолбэком на колонку
   «Регион» из «Статус по отгрузкам», если заказ не нашёлся.
3. **⏱️ SLA: план vs факт** (доп. функционал) — сравнивает плановую дату отгрузки
   (лист «Статус по заказам») с фактической (лист «Статус по отгрузкам») по заказам `1M-...`.
   Если колонка «Дата отгрузки план» пустая — берётся дата раздела
   (`01/07/2026 (заказы на ср)`), т.к. в вашем файле план часто указан только там.

Глобальные фильтры в сайдбаре: период и фильтр «только заказы 1M-...».
Во всех таблицах есть кнопка «Скачать CSV».

## Как запустить локально

```bash
pip install -r requirements.txt
streamlit run app.py
```

Откроется на `http://localhost:8501`.

## Источник данных — только Google Sheet

Приложение скачивает таблицу напрямую по ссылке через
`.../export?format=xlsx`. Для этого таблица должна быть открыта
«Всем, у кого есть ссылка → Читатель» (Файл → Настройки доступа
в самой Google Таблице).

В сайдбаре просто вставьте ID таблицы (кусок ссылки между `/d/` и `/edit`,
по умолчанию уже подставлен ваш) — данные подтянутся автоматически.
Кэш держится 5 минут, кнопка «Загрузить / обновить данные» сбрасывает
кэш и тянет свежие данные немедленно.

> Если таблица приватная (доступ только по приглашению), автозагрузка не
> сработает — Google вернёт ошибку доступа. В этом случае откройте доступ
> по ссылке (см. выше) — это самый простой вариант без настройки
> отдельной авторизации (OAuth/service account).

## Деплой на Streamlit Cloud — если раньше падала установка зависимостей

В requirements.txt версии теперь жёстко зафиксированы (==), это
исправляет типичную ошибку "Error installing requirements" из-за
несовместимых версий при разрешении зависимостей:

```
streamlit==1.38.0
pandas==2.2.2
openpyxl==3.1.2
requests==2.32.3
```

Если после этого всё ещё падает — на странице ошибки нажмите
"Manage app", в терминале будет видна конкретная причина (например,
неверный Python в настройках приложения — тогда в "Advanced settings"
при деплое выставьте Python 3.11 или 3.12).

## Если структура листов немного другая

Вся логика парсинга — в `data_parsing.py`. Если поменяется порядок/название
колонок в исходнике, достаточно поправить номера колонок в
`parse_shipments()` и `parse_orders_status()` — остальной код (UI, агрегации)
трогать не нужно.

## Деплой (чтобы был постоянный URL, не только локально)

Проще всего — [Streamlit Community Cloud](https://streamlit.io/cloud):
залить эти файлы в GitHub-репозиторий и подключить репозиторий там,
бесплатно и с постоянной ссылкой.
